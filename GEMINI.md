🛠 Контекст разработчика (Developer Context)

  1. Философия кода и стандарты
   * Asynchronous First: Весь ввод-вывод (IO) должен быть строго асинхронным. Избегать блокирующих вызовов в основном цикле. При работе с SQLite (текущая
     реализация sqlite3) операции должны быть максимально быстрыми, либо стоит рассмотреть переход на aiosqlite.
   * Type Hinting: Обязательное использование аннотаций типов для всех функций и методов. Это критично для стабильности бота и поддержки IDE.
   * Aiogram 3 Паттерны:
       * Использовать Router для группировки хендлеров вместо регистрации в main().
       * Активно применять Magic Filter (F.text, F.data) и кастомные фильтры.
       * Соблюдать иерархию Middleware (внешние vs внутренние).

  2. Архитектура и структура
   * Separation of Concerns: Логика БД в database_sqlite.py, бизнес-логика в utils.py или отдельных сервисах, UI (тексты/кнопки) — в keyboards.py или
     константах.
   * FSM (Finite State Machine): Для любых многошаговых диалогов (как добавление VPN) использовать State и StatesGroup.
   * Error Handling: Каждое сетевое взаимодействие (API Marzban, Web Panel) должно быть обернуто в try/except с логированием через кастомный logger.py.

  3. Безопасность и мониторинг
   * Access Control: Строгая проверка прав через AdminCheckMiddleware. Чувствительные команды (системные /sh, рестарт) доступны только PRIMARY_ADMIN.
   * Secret Management: Никаких токенов в коде. Все настройки только через .env и config.py.
   * System Integrity: Продолжать развитие security_monitor.py. Логирование атак и нагрузки — приоритет для стабильности сервера.

  4. UI/UX стиль (Dunduk-style)
   * Соблюдать установленный «вайб»: использование специфических эмодзи (▀ Ĺ̯▀ ), (⌐■_■), ASCII-артов для отмены событий.
   * Автоматическое удаление сообщений пользователя и обновление сообщений бота для поддержания чистоты чата (паттерн _update_bot_message).

  5. Масштабирование
   * При добавлении новых API (FastAPI) использовать Pydantic-схемы для валидации входящих данных.
   * Для фоновых задач (ремайндеры) использовать apscheduler или продолжать использовать asyncio.create_task, обеспечивая механизм обработки исключений
     внутри задач, чтобы они не «падали» молча.
